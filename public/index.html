Structured memory, not transcript replay.

Raw messages, documents, events, and tool outputs are input. UML turns useful context into structured memory objects like memory items, entities, events, edges, profiles, receipts, and indexes.

Chat history is not memory.

Messages are source material. UML keeps the durable meaning that should survive beyond a single exchange.

Memory is structured meaning.

UML stores verified entities, facts, events, relationships, rollups, and receipts so systems can reason over them later.

Backend is the authority, not the LLM.

The LLM proposes; UML verifies; UML writes. Recall and extraction stay separate.

One graph, many inputs.

Chat is one input type. Not the whole product. UML can ingest useful context from tools, apps, agents, docs, events, workflows, and custom schemas.

Recall selectively. Not by dumping the graph into the prompt.

UML assembles bounded recall for the current turn, task, or workflow using scopes, filters, recency, topic and entity relevance, and memory confidence.

Prompt-ready context

Connected systems receive compact context shaped for the work at hand.

Not raw graph JSON

Recall is bounded and selective, not a dump of every node and edge.

Separate write and read paths

Extraction decides what becomes memory. Recall decides what is relevant now.

Use UML the way your stack works.

API, SDK, MCP, and dashboard workflows all point at the same private structured memory graph.

API

Your app sends turns, events, documents, tool outputs, and workflow state directly to UML.

SDK

Use lifecycle helpers like beforeReply() and afterReply() to automate recall and write paths inside your app.

MCP

UML exposes memory tools to compatible clients, but the client or host model decides when to call them.

Dashboard

View, search, manage, and control your memory graph.

Honest MCP note

When UML is connected through an MCP-capable AI client, UML provides memory tools. The client or host model decides when to call them. If you need guaranteed per-turn capture or recall, use the UML API or SDK inside your app or agent runtime.

A graph that can update, not just append.

New evidence can create, update, promote, invalidate, or supersede memory. UML is not an append-only chat log.

Private by default. Inspectable by design.

Your memory graph belongs to your account. Each connected tool gets scoped access. You can inspect, search, revoke, reset, and control what gets saved.

Privacy Policy

UML does not sell user data. UML does not use user memory for unrelated purposes. User memory belongs to their account. Connected tools access memory only through private links/tokens. Users can revoke links. Users can reset memory. Token secrets/full private links are shown only once where applicable.

Terms & Conditions

Use UML responsibly. User controls their account/memory. Connected tools are third-party services. User is responsible for keeping private links/tokens safe. UML may change features during early development.
